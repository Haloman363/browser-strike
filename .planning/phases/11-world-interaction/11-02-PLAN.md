# Phase 11: World Interaction - Plan 02: Destructible Objects

**Objective:** Implement destructible environmental objects (specifically crates) that react to gunfire and explosives with health-based destruction and visual debris.

## Key Goals
1. **Health Tracking**: Add `health` and `isDestructible` flags to objects in `Factory.js` and `Maps_v2.js`.
2. **Damage Logic**: Update `WeaponSystem.js` and `main.js` (explode logic) to apply damage to environment objects.
3. **Shatter Effect**: Implement an effect that replaces the solid crate with smaller physics-driven debris or simply removes it with a particle burst.
4. **Network Sync**: Ensure destruction is synchronized across all clients (Host-authoritative).

## Proposed Changes

### `src/Factory.js`
- Update `createCrate()` to accept `health` (default e.g., 100) and set `userData.isDestructible = true`.

### `src/systems/WeaponSystem.js`
- In `handleHit()`, check if `hitPart.userData.isDestructible` is true.
- Emit `entity:damage` or similar event for environmental objects.

### `main.js`
- Update `explode()` logic to damage destructible objects within the blast radius.
- Implement a `destroyObject(obj)` function that:
    - Plays a shatter sound.
    - Triggers an explosion/debris particle effect in `FXSystem`.
    - Removes the object from the scene and `objects` array.
- Add host-side notification to sync destruction to clients.

## Verification Plan
1. **Manual Test**:
    - Shoot a crate until it breaks. Verify it disappears and plays a sound/effect.
    - Throw an HE grenade near crates and verify they are destroyed.
    - Verify that destroyed crates no longer provide collision.
2. **Multiplayer Test**:
    - Ensure that when the host destroys a crate, it is also destroyed on the client's screen.
