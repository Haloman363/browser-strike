# Phase 11: World Interaction - Plan 01: Ladder Mechanics

**Objective:** Implement functional ladder mechanics including auto-attach, look-based climbing, and dismount logic, with integrated combat penalties.

## Key Goals
1. **Detection & Attachment**: Implement proximity-based auto-attach logic in `PlayerControllerSystem.js`.
2. **Look-Based Climbing**: Override gravity and map movement inputs to vertical velocity based on camera pitch while on a ladder.
3. **Dismount Logic**: Handle jumping off or moving away from the ladder plane to exit the climbing state.
4. **Combat Penalty**: Apply a massive spread penalty in `WeaponSystem.js` when the player is climbing.
5. **Map Integration**: Define ladder objects in `Maps_v2.js` or `main.js` for testing.

## Proposed Changes

### `src/systems/PlayerControllerSystem.js`
- **State**: Add `this.isOnLadder = false`.
- **Update Loop**:
    - Add `checkLadders()`: Proximity check against objects with `userData.type === 'ladder'`.
    - Modify `updateVelocity()`:
        - If `isOnLadder`:
            - Disable gravity.
            - `velocity.y = (forward - backward) * speed * -Math.sin(camera.rotation.x)`.
            - Handle dismount on jump or distance threshold.
- **Physics**: Ensure collision doesn't interfere with climbing (e.g., ignore ladder collisions while climbing or treat them as triggers).

### `src/systems/WeaponSystem.js`
- **Spread Logic**: Check `playerController.isOnLadder` and apply a multiplier (e.g., 5.0x) to the base spread.

### `src/Maps_v2.js` (or `main.js` for quick testing)
- Add a few ladder entities (thin boxes with `userData: { type: 'ladder' }`).

## Verification Plan
1. **Automated Tests**: Add unit tests in `PlayerControllerSystem.test.js` to verify `isOnLadder` state transitions.
2. **Manual Test**:
    - Walk into a ladder and verify attachment.
    - Look up and press W to climb up.
    - Look down and press W to climb down.
    - Press Space to jump off.
    - Fire a weapon while climbing and verify increased spread.
