# Phase 14: Gameplay Mechanics - Plan 03: Hostage Rescue Game Mode

**Objective:** Implement the classic Hostage Rescue game mode, featuring interactive hostage NPCs, rescue zone logic, and associated tactical rules.

## Key Goals
1. **Hostage Entity**: Create a procedural `Hostage` model in `CharacterFactory.js` and add a `Hostage` class/system to manage behavior.
2. **Follow Logic**: Implement a simple follow-AI where the hostage trails the rescuing player (CT) with a distance buffer.
3. **Rescue Zone**: Define rescue zones in `Maps_v2.js` (typically at CT spawn) that trigger a round win upon hostage arrival.
4. **Economic Consequences**: Implement penalties for harming hostages and rewards for rescuing them.
5. **UI & VO**: Add "Hostage Rescued" alerts and on-screen tracking.

## Proposed Changes

### `src/factory/CharacterFactory.js`
- Add `createHostageModel()`: A unique variant of the humanoid model (e.g., orange jumpsuit or civilian clothes).

### `src/systems/HostageSystem.js` (New System)
- Manages all active hostages.
- Handles "E" interaction to start/stop leading.
- Manages following physics (moving towards player if distance > threshold).
- Detects if hostage is in a `RescueZone`.

### `src/Maps_v2.js`
- Define `rescueZones` in map data.
- Place hostages in specific maps (e.g., Mirage, or a new small test map).

### `src/systems/RoundSystem.js`
- Add `hostage:rescued` win condition.
- CT win if all/one hostage rescued.

### `main.js`
- Update interaction logic to support hostages.

## Verification Plan
1. **Interaction Test**: Press "E" near a hostage and verify they start following.
2. **Pathing Test**: Lead the hostage through corners and verify they don't get stuck (within reasonable limits of simple follow AI).
3. **Win Test**: Lead a hostage to the rescue zone and verify CTs win the round.
4. **Economy Test**: Shoot a hostage and verify money is deducted.
