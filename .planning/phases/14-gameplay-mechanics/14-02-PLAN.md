# Phase 14: Gameplay Mechanics - Plan 02: Refined Movement

**Objective:** Implement high-skill movement mechanics including counter-strafing for faster accuracy recovery and damage-based tagging to penalize players caught in the open.

## Key Goals
1. **Counter-Strafing**: Detect opposite key presses during movement and apply an instant deceleration impulse to reset velocity to 0.
2. **Movement Tagging**: Temporarily reduce `MOVE_SPEED` when a player takes damage, based on the weapon's power.
3. **Accuracy Integration**: Ensure weapon spread correctly reflects the "standing still" state immediately after a successful counter-strafe.
4. **Crouch-Jumping**: Refine the jump logic to allow slightly higher vertical clearance when crouching mid-air.

## Proposed Changes

### `src/systems/PlayerControllerSystem.js`
- **Counter-Strafing**:
    - In `applyInput()`, track previous input state.
    - If `KeyA` is pressed while `velocity.x` is positive (moving right), or `KeyD` while `velocity.x` is negative, set `velocity.x = 0`.
    - Apply same logic for `KeyW`/`KeyS` on `velocity.z`.
- **Tagging**:
    - Add `this.taggingFactor = 1.0`.
    - Add `applyTagging(intensity)` method.
    - In `applyInput()`, multiply `speed` by `this.taggingFactor`.
    - Recover `taggingFactor` back to 1.0 over time (e.g., 500ms).

### `main.js`
- In `takeDamage()`, notify `PlayerControllerSystem` to apply tagging.

### `src/systems/WeaponSystem.js`
- Verify that `calculateSpread()` uses the zeroed-out velocity from counter-strafing instantly.

## Verification Plan
1. **Feel Test**: Use a "Speed/Accuracy" debug display to verify that counter-strafing resets velocity to 0 significantly faster than natural friction.
2. **Tagging Test**: Stand in front of a bot (or a Molotov) and verify movement speed drops upon taking damage.
3. **Consistency Test**: Ensure counter-strafing doesn't feel "jittery" or cause desync in multiplayer.
