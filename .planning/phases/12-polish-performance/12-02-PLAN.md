# Phase 12: Polish & Performance - Plan 02: Build & Asset Optimization

**Objective:** Reduce the initial bundle size and improve asset management for a faster v1 launch.

## Key Goals
1. **Module Splitting**: Divide the massive `Factory.js` into smaller, specialized modules.
2. **Vite Chunking**: Configure `vite.config.js` to group related modules into separate chunks (e.g., character logic, weapon logic).
3. **Texture Loading Strategy**: Add a way to toggle high-res texture loading to avoid downloading 50MB+ of data on slow connections.
4. **Final Polish**: Verify all UI elements and HUD responsiveness.

## Proposed Changes

### `src/factory/` (New Directory)
- `FactoryUtils.js`: Shared builder class and base functions.
- `WeaponFactory.js`: All weapon-specific procedural logic.
- `CharacterFactory.js`: All character/bot procedural logic.
- `EnvironmentFactory.js`: Wall, crate, pillar, and ladder logic.

### `vite.config.js`
- Update `build.rollupOptions.output.manualChunks` to split `three.js`, `factory`, and `ui` logic.

### `src/TextureGenerator.js`
- Add a `lowRes` flag or setting to `GameState`.
- If `lowRes` is true, skip the `loader.load()` call for external high-res PNGs and stick to the procedural fallback.

### `main.js`
- Update imports to use the new factory modules.

## Verification Plan
1. **Build Size Test**: Compare the size of `main.js` and total assets before and after refactoring.
2. **Loading Test**: Verify that disabling high-res textures results in zero network requests for large PNGs.
3. **Smoke Test**: Ensure all weapons and character models still render correctly after the module split.
