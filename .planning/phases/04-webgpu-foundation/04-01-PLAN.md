<task type="auto">
  <name>Task 2: Refactor Engine.js for WebGPURenderer</name>
  <files>src/core/Engine.js</files>
  <action>
    Migrate the core renderer from `WebGLRenderer` to `WebGPURenderer` and implement asynchronous initialization.
    
    Implementation details:
    - Import `WebGPURenderer` from `three/webgpu` instead of using the standard `THREE.WebGLRenderer`.
    - Change `init(options = {})` to `async init(options = {})`.
    - Replace `this.renderer = new THREE.WebGLRenderer(...)` with `this.renderer = new WebGPURenderer({ antialias: true })`.
    - Call `await this.renderer.init()` within the `init` method.
    - Add `this.backend = this.renderer.isWebGPU ? 'WebGPU' : 'WebGL'` property to the engine for later use in the HUD.
    - Ensure `onResize` and `loop` still work with the new renderer instance.
  </action>
  <verify>
    <automated>npm run build</automated>
  </verify>
  <done>Engine.js uses WebGPURenderer with async initialization.</done>
</task>

<task type="auto">
  <name>Task 3: Update Entry Points for Async Initialization</name>
  <files>main.js, src/DevViewer.js</files>
  <action>

    - Clean up any legacy WebGL-specific initialization code that is now handled by `engine.init()`.
    
    Implementation details in `src/DevViewer.js`:

  </action>
