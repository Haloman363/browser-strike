# Phase 15: Asset Integration - Plan 02: External Character Integration

**Objective:** Integrate high-fidelity external character models for players and bots, starting with the Arctic Avenger.

## Key Goals
1. **Character Loading**: Update `CharacterFactory.js` to support asynchronous loading of external models using `AssetManager`.
2. **Arctic Avenger Integration**: Map the Arctic Avenger OBJ/MTL assets to the `TERRORIST` team.
3. **Rigging Alignment**: Ensure external models are correctly positioned, scaled (e.g., 18 units high), and oriented to match the engine's animation and collision system.
4. **Team-Based Loading**: Enable separate external assets for CT and T teams while maintaining procedural fallbacks.

## Proposed Changes

### `src/factory/CharacterFactory.js`
- Import `assetManager`.
- Update `createHumanoidModel(team)`:
    - If `highRes` is enabled and an external asset exists for the `team`:
        - Create procedural proxy group.
        - Trigger `AssetManager.loadOBJ`.
        - On load: Replace children of proxy group.
        - Apply scaling (likely 0.5x - 1.0x depending on source) and Y-offset to align feet with ground.

### `main.js`
- Ensure `highResTextures` setting is correctly passed or read within the factory.

## Verification Plan
1. **Visual Test**: Start a Deathmatch and verify that T bots render as Arctic Avengers instead of procedural humanoids.
2. **Animation Test**: Verify that basic leg swinging (if implemented via group rotation) still functions or gracefully fails for external models.
3. **Collision Test**: Verify that headshot hitboxes still align with the high-res character's head.
