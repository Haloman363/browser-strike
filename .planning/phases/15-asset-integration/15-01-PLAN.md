# Phase 15: Asset Integration - Plan 01: External Weapon Loading

**Objective:** Implement an asynchronous asset management system and integrate the first high-fidelity external weapon model (Desert Eagle).

## Key Goals
1. **AssetManager**: Create a centralized class in `src/core/AssetManager.js` to handle `GLTFLoader` and `OBJLoader` with caching.
2. **Proxy-Swap Logic**: Update `WeaponFactory.js` to return a procedural placeholder immediately while asynchronously loading and swapping in the external asset.
3. **Desert Eagle Integration**: Hook up the high-res Desert Eagle OBJ/MTL files located in `public/assets/models/weapons/deagle/`.
4. **Bandwidth Awareness**: Respect the `highResTextures` setting to skip external asset loading if disabled.

## Proposed Changes

### `src/core/AssetManager.js` (New File)
- Singleton class managing `THREE.LoadingManager`.
- Methods: `loadGLB(url)`, `loadOBJ(url, mtlUrl)`, `getCache(url)`.

### `src/factory/WeaponFactory.js`
- Update `createGunModel()`:
    - If `highRes` is enabled and an external asset exists for the `weaponKey`:
        - Create procedural proxy group.
        - Trigger `AssetManager.load...`.
        - On load: Replace children of the proxy group with the external mesh.
        - Apply necessary scaling/rotation to align with procedural standards.

### `main.js`
- Initialize `AssetManager` and provide it to the `Engine` context.

## Verification Plan
1. **Visual Test**: Equip the Desert Eagle and verify it transitions from procedural to high-res (or loads high-res instantly if cached).
2. **Settings Test**: Toggle "High-Res Textures" to OFF and verify the Desert Eagle remains procedural.
3. **Multiplayer Test**: Ensure external models do not interfere with hit registration or synchronization.
