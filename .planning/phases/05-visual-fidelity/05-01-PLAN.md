<task type="auto">
  <name>Task 2: Convert Factory Materials to PBR (Wave 1)</name>
  <files>src/Factory.js</files>
  <action>
    Update Factory.js to use PBR properties:
    1. Change all instances of `MeshPhongNodeMaterial` to `MeshStandardNodeMaterial`.
    2. Remove `shininessNode` assignments.
    3. Add `roughnessNode` and `metalnessNode` using TSL `float()`.
    
    Values to assign:
    - Metal (Gun body, slide, blade, metal parts): roughness 0.2, metalness 1.0.
    - Polymer (Grips, tactical gear): roughness 0.8, metalness 0.0.
    - Skin: roughness 0.6, metalness 0.0.
    - Cloth/Wood: roughness 0.9, metalness 0.0.
    
    Ensure `colorNode` and `textureNode` assignments remain intact.
  </action>
  <verify>
    <automated>npm run build && npm test src/Factory.test.js</automated>
  </verify>
  <done>All materials in Factory.js use MeshStandardNodeMaterial with PBR properties. Tests pass.</done>
</task>

<task type="auto">
  <name>Task 3: Implement Post-processing and CRT Effect (Wave 2)</name>
  <files>src/core/Engine.js</files>
  <action>
    Integrate the PostProcessing pipeline:
    1. Import `PostProcessing` and `pass` from `three/webgpu` (or the appropriate addon path if using Three.js standard distribution).
    2. In `Engine.init`, create `this.postProcessing = new PostProcessing(this.renderer)`.
    3. Define the effect chain:
       - `const scenePass = pass(this.scene, this.camera)`
       - `const crtPass = scenePass.film(time, 0.5, 512, false)` or implement a custom TSL CRT shader if `film` is not directly available on the pass.
       - Note: Use `this.clock.elapsedTime` or a TSL `timerLocal` for the film grain animation.
    4. Set `this.postProcessing.outputColorNode = crtPass`.
    5. Modify `Engine.loop` to call `this.postProcessing.render()` instead of `this.renderer.render(this.scene, this.camera)`.
  </action>
  <verify>
    <automated>npm run build</automated>
  </verify>
  <done>Engine uses PostProcessing for rendering. CRT effect is integrated into the output.</done>
</task>
