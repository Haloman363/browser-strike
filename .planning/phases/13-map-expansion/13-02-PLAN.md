# Phase 13: Map Expansion - Plan 02: Inferno Layout & Geometry

**Objective:** Implement a tactical representation of the Inferno map using procedural geometry, introducing an `createArch` tool for the map's iconic architecture.

## Key Goals
1. **Procedural Arches**: Add `createArch()` to `EnvironmentFactory.js` to support the curved tunnels and gateways characteristic of Inferno.
2. **Inferno Blocking**: Define the core layout of Inferno (A Site, B Site, Mid, Banana, Apartments) in `Maps_v2.js`.
3. **Scale Consistency**: Maintain the 1 unit = 1 inch scale for consistent tactical timings.
4. **Vertical Complexity**: Use `createStairs` and `createLadder` for Apartments and Pit areas.

## Proposed Changes

### `src/factory/EnvironmentFactory.js`
- Implement `createArch(width, height, thickness, depth, x, y, z, ry, scene, objects)`:
    - Generates a curved archway using a series of rotated boxes or an extruded tube segment.
    - Adds `userData.isSolid = true`.

### `src/Maps_v2.js`
- Add `inferno` to the `Maps` object:
    - **Spawn Points**: CT Spawn (near back alley), T Spawn (near bridge).
    - **A Site**: Pit, Graveyard, Balcony, Boiler.
    - **Mid**: T Ramp, Top Mid, Arch Side, Library.
    - **B Site**: Banana, Sandbags, Fountain, Coffins.
- Integrate `createArch` for the iconic "Arch" connecting Mid to A site.

## Verification Plan
1. **Layout Audit**: Verify Banana and Apartments provide correct tactical cover and sightlines.
2. **Collision Test**: Ensure smooth movement through archways and up to Balcony.
3. **Performance Check**: Monitor draw calls with the addition of more complex arch geometry.
