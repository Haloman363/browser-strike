# Phase 13: Map Expansion - Plan 03: Global Map Optimization & Prop Dressing

**Objective:** Optimize rendering performance for large map layouts through static geometry merging and enhance visual identity with procedural detail props.

## Key Goals
1. **Static Geometry Merging**: Implement a system to merge static map geometry (walls, floors) by material to reduce draw calls.
2. **Procedural Dressing**: Add `createPipe()` and `createLamp()` to `EnvironmentFactory.js` for enhanced map detail.
3. **Dust 2 Refinement**: Update the legacy Dust 2 blockout with new tools (`createStairs`, `createArch`) and refined proportions.
4. **Collision Audit**: Ensure merged geometry maintains accurate per-object collision.

## Proposed Changes

### `src/Utils.js`
- Implement `mergeGeometry(meshes)`: Utility to combine multiple meshes into a single `BufferGeometry` for performance.

### `src/factory/EnvironmentFactory.js`
- Implement `createPipe(radius, length, x, y, z, rotation, scene, objects)`: Industrial detail.
- Implement `createLamp(x, y, z, color, scene)`: Add point lights and procedural fixtures.

### `src/Maps_v2.js`
- Refactor map `build` functions to use a "Merge Phase" at the end of construction.
- Update `dust2` to use `createStairs` for Short/Ramp and `createArch` for Tunnels.

### `main.js`
- Ensure map loading logic correctly handles the transition to merged geometry.

## Verification Plan
1. **Draw Call Audit**: Use `renderer.info` or Chrome DevTools to verify a significant reduction in draw calls on full map builds.
2. **Visual Audit**: Ensure all detail props render correctly and textures are not distorted by merging.
3. **Collision Test**: Verify that "Ghost Walls" or missing collision do not occur after merging.
