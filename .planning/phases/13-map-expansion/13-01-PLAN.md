# Phase 13: Map Expansion - Plan 01: Mirage Layout & Geometry

**Objective:** Implement a tactical representation of the Mirage map using procedural geometry, including a new `createStairs` tool for better verticality.

## Key Goals
1. **Procedural Stairs**: Add `createStairs()` to `EnvironmentFactory.js` to support slanted vertical movement.
2. **Mirage Blocking**: Define the core layout of Mirage (A Site, B Site, Mid, Jungle, Palace) in `Maps_v2.js`.
3. **Scale Calibration**: Ensure the map dimensions align with the player's movement speed for an authentic CS feel.
4. **Interactive Elements**: Place ladders and destructible crates in tactically significant locations.

## Proposed Changes

### `src/factory/EnvironmentFactory.js`
- Implement `createStairs(width, height, depth, steps, x, y, z, rotation, scene, objects)`:
    - Generates a series of stepped boxes or a single slanted wedge with invisible "step" collision.
    - Adds `userData.type = 'stairs'`.

### `src/Maps_v2.js`
- Add `mirage` to the `Maps` object:
    - **Spawn Points**: CT Spawn (near ticket), T Spawn (near palace/apps).
    - **A Site**: Ticket booth, stairs, palace entrance, pillars.
    - **Mid**: Window, connector, catwalk, top mid.
    - **B Site**: Apartments, van, bench, market.
- Use `createStairs` for Connector, Jungle, and Apartments.

### `src/Constants_v2.js`
- Verify `MOVE_SPEED` and `JUMP_FORCE` relative to the new map scale.

## Verification Plan
1. **Geometry Test**: Verify all areas of Mirage are accessible and collision works as expected.
2. **Timing Test**: Measure "Rush" timings (e.g., T Spawn to A Site) to ensure tactical balance.
3. **Visual Test**: Ensure textures are correctly tiled on new stair geometry.
